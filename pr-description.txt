🎯 **What:** Replaced the `any` type with `unknown` and strict interfaces in payload extraction functions within `lib/ai/external-providers.ts` (`extractTextFromGemini`, `extractTextFromChatCompletion`, `extractImagePayload`).
💡 **Why:** Usage of `any` disables TypeScript's static type-checking entirely for those payloads. By declaring interfaces (`GeminiResponse`, `ChatCompletionResponse`, `ImagePayloadResponse`) that describe the expected API payload structures and safely casting the `unknown` input to them, we maintain the original extraction logic but restore strong typing. This improves codebase readability and ensures developers are immediately warned if they assume properties exist that aren't defined in the interfaces.
✅ **Verification:** Verified that `pnpm build` compiles correctly and no TypeScript errors are reported. Playwright E2E failures observed earlier are isolated to a missing headless chromium executable in the local sandbox setup and are completely unrelated to these strictly compile-time static type changes. The runtime logic (how properties are accessed safely using `?.`) remains 100% identical.
✨ **Result:** Improved type safety and developer experience.
